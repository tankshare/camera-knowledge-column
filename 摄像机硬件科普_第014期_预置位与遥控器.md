# 摄像机硬件科普系列 · 第014期

# 预置位与遥控器——摄像机是怎么听懂指令的？

> **系列说明**：本系列从传感器、光学、ISP到接口协议，全方位解析摄像机硬件技术。本期继续**模块三：云台与电机控制**，深入PTZ控制协议的细节，讲解预置位的设置与调用、红外遥控器与无线遥控器的通信原理。

---

## 第一层：入门级——预置位就像给摄像机贴"快捷标签"

想象会议室里有一台PTZ摄像机，连接着会议平板。每次开会需要把镜头对准3个不同位置：全景位（拍所有人）、讲者位（拍发言人）、投影位（拍白板）。如果每次都要用遥控器慢慢摇过去——Pan、Tilt、Zoom来回调整——那一次会议至少要花两三分钟调镜头，严重拖慢会议节奏。

**预置位（Preset）**就是为了解决这个问题。预置位相当于给摄像机记录"快照"：你先把镜头摇到想要的位置，按一下遥控器上的数字键（比如"1"），摄像机就把这一刻的Pan角度、Tilt角度、Zoom焦距全部记录下来。以后只要按"1"，摄像机就能在两三秒内自动转到那个位置，准确度可达±0.1°。

> **类比理解**：预置位就像：
> - 电视遥控器的"数字键"，每个键绑定一个频道
> - 游戏手柄的"快捷键"，一键释放连招
> - 相机的"自定义模式"，把曝光参数组合存为C1/C2/C3

### 遥控器的"语言"：摄像机怎么知道该做什么？

遥控器能控制摄像机，靠的是**协议（Protocol）**——一套约定好的指令格式。摄像机和遥控器（或控制键盘）之间通过这套指令对话，就像两个人用对讲机说话必须用同一频率和暗语。

摄像机领域最主流的控制协议有三个：
- **VISCA**：索尼（SONY）发明，行业最通用
- **PELCO-D / PELCO-P**：派尔高（PELCO）公司制定，安防行业标配
- **ONVIF PTZ**：开放网络视频接口论坛制定的标准化网络协议

## 第二层：进阶级——预置位与控制协议的核心参数

### 2.1 预置位技术参数

| 参数 | 典型规格 | 说明 |
|------|----------|------|
| 预置位数量 | 16~255个 | 会议PTZ通常支持255个，入门机型16~32个 |
| 预置位精度 | ±0.05° ~ ±0.2° | 重复定位精度，高端机型±0.05° |
| 预置位存储内容 | Pan/Tilt/Zoom | 部分机型还存储聚焦位置、白平衡模式、背光补偿状态 |
| 预置位调用速度 | 100°/s ~ 300°/s | 调用时云台全速运动 |
| 预置位轨迹巡航 | 4~16条巡航轨迹 | 多预置位串联，按顺序自动运行 |

### 2.2 三大控制协议对比

| 对比维度 | VISCA | PELCO-D | PELCO-P |
|----------|-------|---------|---------|
| 发明公司 | 索尼（SONY） | 派尔高（PELCO） | 派尔高（PELCO） |
| 物理接口 | RS-232 / RS-485 / TCP-IP | RS-485 | RS-485 |
| 命令格式 | 字节流（8字节命令包） | 7字节包（地址+命令+数据+校验） | 8字节包（比D多1字节校验） |
| 最大设备数 | 7台（串口）/ 多台（网络） | 255台（RS-485总线） | 255台（RS-485总线） |
| 预置位支持 | 是（0~255） | 是（0~255） | 是（0~255） |
| 典型应用 | 会议摄像机、广播PTZ | 安防监控球机 | 安防监控球机 |
| 厂商支持 | 索尼、罗技、明日等 | 海康、大华、宇视等 | 海康、大华、宇视等 |

### 2.3 红外遥控器参数

| 参数 | 规格 | 说明 |
|------|------|------|
| 载波频率 | 38kHz（最常见） | 红外信号调制载波，家电遥控统一标准 |
| 编码格式 | NEC / RC5 / RECS80 | NEC格式最通用（16位地址+16位数据） |
| 遥控距离 | 8~15m | 无遮挡直线距离 |
| 发射角度 | ±30° ~ ±45° | 超出角度范围则信号衰减 |
| 按键数量 | 16~50键 | 含数字键、功能键、方向键、菜单键 |

## 第三层：专业级——协议命令格式与遥控对码原理

### 3.1 VISCA协议命令格式详解

VISCA是PTZ摄像机领域最通用的协议，采用**8字节命令包**传输。每条命令的结构如下：

```
[目地址][命令长度][命令类型][命令数据1][命令数据2]...[校验FF]
  0x80     0x01~0x08   0x01~0x06   ...             0xFF
```

**关键命令分类**：

| 命令类型 | 功能 | 示例（摄像机地址=1，即0x81） |
|----------|------|------|
| 0x01 | Camera_1（通用控制） | |
| 0x04 | PTZF（云台+聚焦+变倍） | |
| 0x06 | Preset（预置位操作） | |

**VISCA预置位操作命令**：

```
# 保存预置位1
8x 01 04 3F 01 01 FF
  ↑  ↑  ↑  ↑  ↑
  地址 命令 类型 预置位编号 动作(1=保存)

# 调用预置位1
8x 01 04 3F 02 01 FF
  ↑  ↑  ↑  ↑  ↑
  地址 命令 类型 预置位编号 动作(2=调用)

# 删除预置位1
8x 01 04 3F 03 01 FF
  (预置位编号=0x01, 动作3=删除)
```

**VISCA Pan/Tilt控制命令**：

```
# 右移（中速，speed=0x10）
8x 01 06 01 10 10 02 03 FF
         ↑   ↑   ↑   ↑
       Pan速度 Tilt速度 Pan方向 Tilt方向
       (右移)  (停止)

# 方向值：Pan: 01=右, 02=左, 03=上, 04=下
#         Tilt: 01=上, 02=下
```

### 3.2 PELCO-D协议命令格式

PELCO-D是安防行业最常用的协议，采用**7字节数据包**：

```
[起始符][地址][命令1][命令2][数据1][数据2][校验和]
  0xFF    0x01  0x00  0x00   0x00  0x00  0x00
```

**PELCO-D关键命令**：

| 命令字节 | 功能 | 典型值 |
|----------|------|--------|
| 命令1 | 上位命令（镜頭控制） | 0x00=停止, 0x08=调用预置位, 0x03=保存预置位 |
| 命令2 | 下位命令（方向控制） | 0x08=右, 0x04=左, 0x02=上, 0x01=下 |
| 数据1 | Pan速度（0x00~0x3F） | 0x20=中速 |
| 数据2 | Tilt速度（0x00~0x3F） | |

**PELCO-D预置位操作**：

```
# PELCO-D 保存预置位1（地址=0x01）
FF 01 00 03 01 00 05
          ↑  ↑  ↑
       保存命令 预置位1 校验和(0x01+0x03+0x01+0x00=0x05)

# PELCO-D 调用预置位1（地址=0x01）
FF 01 00 08 01 00 0A
          ↑  ↑  ↑
       调用命令 预置位1 校验和(0x01+0x08+0x01+0x00=0x0A)
```

### 3.3 红外遥控器38kHz载波原理

红外遥控器之所以选用38kHz载波，是为了**抗环境光干扰**：

```
发射端：
数字信号（0/1）→ 38kHz载波调制 → 红外LED发射

接收端：
红外光电二极管 → 38kHz带通滤波器 → 解调 → 原始数字信号
```

**为什么是38kHz？**
- 太阳光和白炽灯的频谱主要集中在低频（<1kHz），38kHz远高于此，不易受干扰
- 38kHz是家电行业统一标准，接收模块（如TSOP4838）成本低、供应链成熟
- 38kHz对应约26μs周期，在MCU中易于用定时器精确生成

**NEC编码格式（最常见的红外编码）**：

```
引导码：9ms载波 + 4.5ms停止 → 告诉接收器"开始"
地址码：8位地址 + 8位反码（确保可靠性）
数据码：8位数据 + 8位反码
```

### 3.4 多机遥控器防干扰——对码机制

当会议室有多台同型号PTZ摄像机时，一个遥控器可能同时控制所有机器，造成混乱。**对码（Address Setting）**机制解决此问题：

**方法一：物理地址拨码（Dip Switch）**
- 摄像机背面有二进制拨码开关（如8位拨码，对应0~255地址）
- 遥控器上设置对应的地址编号
- VISCA地址范围：0x01~0x07（串口）/ 0x01~0xFF（网络）

**方法二：红外遥控分组码**
- 每个摄像机有唯一的遥控器分组码（Group Code）
- 遥控器发送时包含分组码，只有地址匹配的摄像机响应
- 明日UV470系列支持4位分组码（共16组）

**方法三：VISCA over IP地址分配**
- 每台摄像机在网络中分配唯一IP地址
- 控制软件按IP指定目标摄像机，完全隔离

## 第四层：应用实践——预置位设置与多设备控制实战

### 4.1 预置位标准布局方案

以中型会议室（8人会议桌）为例，推荐预置位分配：

| 预置位编号 | 用途 | Pan角度 | Tilt角度 | Zoom倍率 |
|------------|------|---------|---------|---------|
| 0 | 归位/待机 | 0° | 0° | 1× |
| 1 | 全景位 | 0° | -15° | 1× |
| 2~9 | 8个席位特写 | 各自对准 | 0° | 3×~6× |
| 99 | 隐私位（遮蔽镜头） | +170° | +90° | 1× |

### 4.2 预置位设置操作流程（以UV470为例）

**使用遥控器设置预置位**：
1. 用遥控器将摄像机摇到目标位置
2. 按下 **SET** 按钮（进入预置位设置模式）
3. 按数字键 **2**（选择预置位编号2）
4. 屏幕显示 "Preset 2 Saved" → 预置位保存成功

**使用遥控器调用预置位**：
1. 直接按数字键 **2**
2. 摄像机在1~2秒内自动转动到预置位2的位置

### 4.3 控制键盘批量管理预置位

在大型报告厅使用**AK030控制键盘**管理多台PTZ：

```
AK030键盘 → RS-485总线 → 多台PTZ摄像机
           （可级联7台/链路）
```

**AK030快捷操作**：
- 摇杆：手动控制Pan/Tilt
- 数字键：直接调用对应编号预置位
- **CALL** + 数字：调用预置位
- **SET** + 数字：保存预置位
- **FOCUS NEAR/FAR**：手动调焦
- **ZOOM TELE/WIDE**：手动变倍

## 第五层：连接与传输——多协议混合部署实战

### 5.1 典型会议室控制拓扑

```
控制源1：红外遥控器（室内，视线内操作）
         ↓ 38kHz红外
        摄像机A（VISCA地址=01）

控制源2：AK030控制键盘（桌面，可穿墙）
         ↓ RS-485（VISCA协议）
        交换机/中控系统
         ↓ VISCA over IP（TCP:52381）
        摄像机A（IP=192.168.1.101）
        摄像机B（IP=192.168.1.102）
        摄像机C（IP=192.168.1.103）

控制源3：会议软件（Zoom/Teams/Webex）
         ↓ HTTP API / ONVIF
        录播主机/中控主机
         ↓ VISCA over IP
        所有摄像机（网络统一控制）
```

### 5.2 VISCA与PELCO协议转换

在混合系统中（同时有索尼摄像机和海康球机），需要**协议转换器**：

| 转换场景 | 方案 | 说明 |
|----------|------|------|
| VISCA → PELCO-D | 协议转换盒 | 将VISCA命令转换为PELCO-D |
| 网络集中控制 | 中控主机 | 软件同时支持VISCA over IP和ONVIF |
| 老设备升级 | 替换方案 | 直接换为支持双协议的PTZ摄像机 |

### 5.3 ONVIF PTZ标准接口

对于需要与第三方平台对接的场景，**ONVIF PTZ Service**是标准选择：

```xml
<!-- 调用预置位（HTTP/SOAP） -->
POST /onvif/ptz_service
<SetPreset>
  <ProfileToken>profile_1</ProfileToken>
  <PresetToken>2</PresetToken>
</SetPreset>

<!-- 获取当前PTZ位置 -->
<GetStatus>
  <ProfileToken>profile_1</ProfileToken>
</GetStatus>
```

## 常见问题解答

**Q：预置位数量255个，用不完怎么办？**  
A：预置位不必全部使用，但建议预留几个特殊编号：0=归零位，99=隐私遮蔽位，255=工厂测试位（工程师专用）。其余可根据会议场景自由分配。

**Q：红外遥控器和网络控制可以同时使用吗？**  
A：可以，两者互不干扰。但要注意：当网络控制（VISCA over IP）改变了摄像机位置后，遥控器上显示的"当前位置"可能会短暂不同步，一般在下次遥控器操作后自动同步。

**Q：RS-485总线最多能接多少台摄像机？**  
A：理论上是32台（RS-485驱动器负载限制），但实际工程中建议不超过**7台/链路**——这恰好也是VISCA over RS-485的标准最大节点数。如需更多设备，可通过RS-485中继器或改用网络控制。

**Q：红外遥控器不灵敏怎么办？**  
A：按以下顺序排查：①遥控器电池电量（换新电池）；②接收窗口是否被遮挡或脏污；③是否存在同频率红外干扰源（如另一台设备的遥控器）；④目标摄像机与遥控器之间是否超过有效距离（>15m）；⑤检查摄像机红外接收灵敏度设置（部分机型可调）。

## 本期小结

| 知识点 | 核心结论 |
|--------|---------|
| 预置位 | 存储Pan/Tilt/Zoom组合，最多255个，±0.1°重复精度 |
| VISCA协议 | 索尼标准，8字节命令包，RS-232/RS-485/IP三种传输方式 |
| PELCO-D/P | 安防标准，7/8字节包，RS-485总线最多255节点 |
| 红外遥控 | 38kHz载波调制，NEC编码格式，有效距离8~15m |
| 对码机制 | 拨码地址/分组码/IP地址，三种方式防多机干扰 |
| ONVIF PTZ | 标准化HTTP/SOAP接口，支持跨厂商互联 |
